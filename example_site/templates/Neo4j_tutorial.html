The graph will not as of right now be highly interesting, but it is more of to show the mechanics of pushing data to a Neo4j graph. I am making the assumption that you already have Neo4j installed and setup on your current machine, or have it properly networked so that you can connect to it from your current machine.

Neo4j-driver¶

I have currently looked at two different packages for working with Neo4j, the first is the basic neo4j-driver package. It seemed to work well, but compared to the next package was a little bit more verbose. The following code demonstrates how to start a session, create an entry, quiery for that entry, and delete that entry or delete all entries.

In [22]:
# Begin with importing the dependencies as follows. 
from neo4j.v1 import GraphDatabase, basic_auth

#Establish a connection to the neo4j database (replace "bolt://localhost:7687" with your db address if different)
#username, and password are string vlaues in the format ("neo4j", "password)
driver = GraphDatabase.driver("bolt://localhost:7687", auth=basic_auth(username, password))

#Once connected create a session that all further actions will utilize. 
session = driver.session()

All actions using this package seem to be initiated by session.run() inside this goes your cipher query followed by a dictionary with keys corresponding to the fields in the quiery and values containing what you want in those fields.The following are examples of how to create nodes using this method. I was unable to easily create relationships this way.

Py2neo¶

At this point in my expierence with neo4j, I am still learning the cipher language. Because of this I prefer py2neo, as it doesn't require writting cypher queries directly, but there is an option to be able to. The following will first add myself to the graph and then add several calendar events scraped from the internet with the appropriate relationships.

In [26]:
from py2neo import authenticate, Graph, Node, Relationship, NodeSelector

# set up authentication parameters
authenticate("localhost:7474", username, password)

# connect to authenticated graph database
graph = Graph("http://localhost:7474/db/data/")
selector = NodeSelector(graph)

As you can see this code is much easier to read and than the previous. If you look at the picture of the graph you can see that it too, just makes sense, with a node that says "Matt" and an arrow that says "WORKS AT" connected to another node "ExoAnalytic Solutions." Attempting to do something similar in a relational database would reslult at best in a table of people that works at ExoAnalytic or two tables with lists of people and lists of companies all with a cryptic primary key that must be interperted by an anlyst or database admin.

In [27]:
#Each node gets saved as a variable before getting added to the graph.
myself = Node("Person", fname='Matt', lname='Camp',
              linkedin='https://www.linkedin.com/in/matthewjcamp/')

mycompany = Node("Company", name='ExoAnalytic Solutions', url='https://exoanalytic.com/')

myself2mycompany = Relationship(myself, "WORKS AT", mycompany, as_a='Algorithm Developer')

#Now we create the node on the graph
graph.create(myself)
graph.create(mycompany)
graph.create(myself2mycompany)
